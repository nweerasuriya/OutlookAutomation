Email Automation — Client Setup

Getting your automation connected

This guide walks you through what we need to link your Microsoft email addresses to the automation system. Choose the option that suits you best below.

🔧
Option A — You set it up, then send the details securely

You'll follow a few steps inside Microsoft's website to create a connection point for the automation. Nothing gets installed on your computer. At the end, you'll send four pieces of information to your developer via 1Password — a secure password manager — so they never travel over email.

⏱ About 20–30 minutes
Part 1 of 3 — Register the app in Microsoft Azure
1
Sign in to the Microsoft Azure portal

Open a web browser and go to portal.azure.com. Sign in with your Microsoft 365 admin account — this is the same login you use to manage your business emails.

ℹ Not sure if you have admin access? You'll know you're in the right place if you can see a dashboard with tiles and services after logging in. If you get an error, contact whoever manages your IT.
2
Open App registrations

Once you're logged in, look for the search bar at the very top of the page. Type App registrations and click on it when it appears in the results.

Search bar at the top → type "App registrations" → click the result

On the App registrations page, click the button that says + New registration.

6
Set the permissions

In the left menu, click on API permissions. Then click + Add a permission. A panel slides in — click on Microsoft Graph, then select Application permissions.

API permissions → + Add a permission → Microsoft Graph → Application permissions

Use the search box to find and tick each of these three permissions:

Mail.Send
Allows sending emails
Files.ReadWrite.All
Allows accessing your OneDrive files
User.Read.All
Allows looking up email addresses

After adding all three, click the button that says Grant admin consent for [your organisation name] and confirm when prompted.

⚠ This final "Grant admin consent" step is essential — without it the permissions won't actually work, even though they appear in the list.

Part 3 of 3 — Send everything securely via 1Password
9
Create a free 1Password account

1Password is a secure password manager that lets you share sensitive information safely. Go to 1password.com and sign up for a free account if you don't already have one.

✓ 1Password encrypts everything you store. Even 1Password themselves cannot read what you share through it. It's one of the most trusted tools for securely sharing credentials.

🤝
Option B — Give your developer temporary access to do it for you

If you'd prefer not to handle technical steps yourself, you can invite your developer as a temporary Guest Administrator on your Microsoft account. They'll set everything up, move the credentials directly to the secure system, and you remove their access straight after. No sensitive information changes hands.

⏱ About 10 minutes on your end
What happens in this option
ℹ You invite your developer as a Guest Admin → they log in to your Azure portal, set up the app, and copy the credentials directly into the secure system → you remove their access. The credentials never get written down or sent anywhere.
Your steps — inviting a Guest Admin
1
Sign in to the Microsoft 365 admin centre

Open your browser and go to admin.microsoft.com. Sign in with your Microsoft 365 admin account — the same one you use to manage email accounts for your organisation.

2
Navigate to Users

In the left-hand menu, look for Users and click it, then click Guest users.

Left menu → Users → Guest users
ℹ If you don't see "Guest users" you may need to go through the Azure Active Directory portal instead. Go to portal.azure.com → Azure Active Directory → Users → + New guest user.
3
Invite your developer as a guest

Click + Invite external user (or + New guest user). Fill in the form:

NameYour developer's name
Email addressYour developer's email address
Message (optional)You can write: "Temporary access for email automation setup"

Click Invite. Your developer will receive an email asking them to accept.

4
Assign the Global Administrator role

Once the invitation is sent, find the guest user you just created in the list and click their name. Then click Assigned roles (or + Add role) and assign them the Global Administrator role.

Click the guest user's name → Assigned roles → + Add assignments → Search "Global Administrator" → Add
⚠ This gives your developer full admin access. Only do this with someone you trust, and make sure to remove the role as soon as they confirm the setup is done (Step 6 below).
5
Create the OneDrive folders

While you wait for your developer to complete the setup, create the folder structure they'll need in OneDrive. Go to onedrive.com and create the following inside your OneDrive:

Top foldere.g. email-automation (note the name)
Inside thatA folder called: csv
Inside thatA folder called: templates
Inside thatA folder called: logs

Let your developer know the name of the top folder and the email address you're logged into OneDrive with. You can send these over regular email or message — they are not sensitive on their own.

6
Wait for confirmation, then remove access

Your developer will tell you when the setup is complete. As soon as they confirm, remove their administrator role — and optionally remove the guest account entirely.

Users → Guest users → Click their name → Assigned roles → Remove Global Administrator

To remove the guest account completely: go back to their user profile and click Delete user.

✓ Once you remove the role (and optionally the account), your developer no longer has any access to your Microsoft systems. The automation itself runs using the app credentials — not their personal login.
